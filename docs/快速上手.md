# 快速上手 — 接一个源并跑通一次

> 目标：5 步内让「采集 → 入库 → LLM 分级 → 门铃推送 + 日刊页」真正跑起来一次。
> 内置的 `twitter_security_list`（Soundwave）开箱即用，不需要外部服务。

## 前置（各一次性）

```bash
megatron migrate       # 建表 / 升级
megatron serve dev     # 启动，默认 http://localhost:8000
```

浏览器打开 <http://localhost:8000> → 跳转登录页。

**首次登录密码**：若未设 `MEGATRON_ADMIN_PASSWORD`，系统首启会**随机生成并打印到启动日志**
（事件 `bootstrap.admin_user_created_with_generated_password`，字段里就是明文），用户名固定 `admin`。
登录后尽快改密。

> 首启还会自动生成并持久化 `MEGATRON_INGEST_TOKEN`（采集源推送用）和
> `MEGATRON_DAY_TOKEN`（日刊页链接用），落在 `/app/data/` 下。

## 第 1 步 · 看看有哪些源 `/ui/data/sources`

```bash
megatron sources list
```

```
SOURCE_ID                    ADAPTER     MANAGED  ENABLED  AUDIENCE
twitter_security_list        bundle_pull yaml     True     personal
```

这个源来自 [`sources/twitter_security_list.yaml`](../sources/twitter_security_list.yaml)。
**源由 YAML 定义，不在 UI 里增删** —— 改 YAML，然后 `megatron sources sync`。

## 第 2 步 · 把数据拉进来

```bash
megatron sources pull twitter_security_list
# Pulled twitter_security_list: ingested=253 duplicated=706
```

Soundwave 每天把日包提交到自己仓库，Megatron 按 cron 去取。
重复拉是安全的：去重键是 `(source_id, external_id)`，拉过的不会重复入库。

在 `/ui/data/collected` 应该能看到数据了。

## 第 3 步 · 配 LLM 提供商 `/ui/providers`

新建 Provider，填 `model`（如 `deepseek/deepseek-chat`）+ `api_key`。保存。

## 第 4 步 · 配推送渠道 `/ui/channels`

新建渠道（钉钉 / 飞书 / Telegram / 企微），填 webhook URL → 点「测试」确认能收到。

> 想先跑通、暂不接群？可以跳过；任务不选渠道也能完成分析，只是不推送。

## 第 5 步 · 建任务并跑一次 `/ui/tasks`

新建任务：

- **数据源**：`twitter_security_list`
- **时间窗口**：`today`（跑不出数据就改 `date` 指定一个有数据的日子）
- **提示词**：选 **`daily_intel_v1`（每日情报分级）**
- **提供商 / 渠道**：选前面配好的

保存后，把它切到「门铃 + 日刊」模式：

```bash
megatron use-day-bundle --module <任务名>
```

点「立即运行」。

## 你会得到什么

**聊天里收到的（门铃，薄）：**

```
⚡ 安全雷达 · 2026-07-10
入库 30 · 强推 3

🔴 1/3  某某漏洞可远程接管本地 agent
你在跑本地 agent
https://x.com/…

——
全日刊：http://localhost:8000/day/2026-07-10?k=…
```

**点进去的（日刊页，厚）：** 全部条目按 `必看 / 推荐 / 速览` 分档，
带「为什么给你」。手机上直接打开，不用登录。

> 强推最多 3 条，是**服务端强制**的：模型就算报了 8 条，也只会推 3 条，
> 其余降级到日刊页，不会丢。

## 接一个新源？

写一个 YAML 就行，不用改代码 —— 见 [使用指南 · 源接入](使用指南.md#源接入)。
