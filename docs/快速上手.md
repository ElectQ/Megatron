# 快速上手 — 添加一个 MCP 源并跑通一次

> 目标:在 5 步内让「MCP 抓取 → LLM 分析 → Webhook 推送」真正跑起来一次。
> 下面以内置 soundwave(stdio 传输)为例,不需要外部 MCP 服务器。

## 前置(各一次性)

```bash
megatron init          # 初始化数据库表
megatron serve dev     # 启动开发服务器,默认 http://localhost:8000
```

浏览器打开 <http://localhost:8000> → 跳转登录页。

**首次登录密码**:若未设置环境变量 `MEGATRON_ADMIN_PASSWORD`,系统首启会**随机生成一个强密码并打印到启动日志**(日志事件 `bootstrap.admin_user_created_with_generated_password`,字段里就有明文密码),用户名固定 `admin`。从日志里取出登录,登录后尽快改密。
> 想跳过随机密码,可预先设 `MEGATRON_ADMIN_PASSWORD`;或用 `megatron seed` 建一个默认 `admin / admin`(仅限本地测试)。

## 第 1 步 · 配一个 LLM 提供商 `/ui/providers`

新建 Provider,填 `model`(如 `deepseek/deepseek-chat`)+ `api_key`,可选 `api_base`、温度、`max_tokens`。保存。

## 第 2 步 · 配一个提示词模板 `/ui/prompts`

新建模板,示例内容:

```
你是情报分析助手。请对以下条目做要点摘要,输出 Markdown 简报。
```

保存。

## 第 3 步 · 配一个 Webhook 渠道 `/ui/channels`

新建渠道(如钉钉),填 webhook URL / 密钥 → 点「测试」确认能收到测试消息。
> 只想先跑通、暂时不接群?可跳过渠道;任务不选渠道也能完成分析,只是不推送。

## 第 4 步 · 添加 MCP 数据源 `/ui/data/sources`

点「添加 MCP 服务器」:

- **服务器名称**:`sw-twitter`(≤32 字符,不能用保留名 `twitter` / `soundwave` / `mcp`)
- **传输方式**:`stdio(本地命令)`
- **命令行**:**留空**(留空即用内置 soundwave 服务器)
  - (可选)自定义:`python -m mcp_servers.soundwave --repo owner/repo --branch master`

先点 **「测试连接」**(保存前测试,不写库),成功应显示:

```
已连接。工具: list_tweets, search_tweets, get_stats, list_available_dates
```

确认无误后点 **「连接 MCP 服务器」** 保存(会自动生成同名的数据源配置)。

## 第 5 步 · 建任务并运行一次 `/ui/tasks`

新建任务:

- **数据源**:选 `sw-twitter`(第 4 步的名字)
- **时间窗口**:先选 `today`(跑不出数据时改 `range` 放宽,或用 `date` 指定有数据的日子)
- **提示词 / 提供商 / 渠道**:选前面配好的
- **定时**:先留空(仅手动运行)

保存后点 **运行**,到 `/ui/runs` 看结果:

- `completed` 且有输入条数 → 全链路通了 ✅
- `completed / 无数据` → 该时间窗口没有新数据,换 `range` / `date` 再试
- `failed` → 打开看 error(连接 / 命令 / 解析问题)。**失败会明确报错**,不会伪装成"无数据"的假成功。

## 排错速查

| 现象 | 处理 |
|------|------|
| 测试连接失败(stdio) | 确认命令能在服务端环境直接跑通(依赖 / 路径);命令行留空用内置服务器 |
| 一直"无数据" | 先看 `/ui/data/collected` 是否已入库;放宽任务的时间窗口 |
| 任务下拉找不到源 | 确认 MCP 源已保存且启用;名字未用保留字 |
| 想定时跑 | 任务里填 cron,如 `0 7 * * *`(每天 07:00 UTC) |

更完整的说明见 [使用指南](使用指南.md)。
