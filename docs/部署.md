# 部署到 VPS

一台能跑 Docker 的 VPS + 域名 `megatron.qshunter.top`。全流程一条 `deploy.sh`。

```text
浏览器 ─https─▶ Caddy(宿主机 :443, 自动 TLS)
                 └─reverse_proxy─▶ 宿主机 :8010 ─(端口映射)─▶ 容器 :8000 (uvicorn)
                                                                    └─ /app/data (卷 megatron_data) = SQLite + 密钥
```

端口说明:容器内 uvicorn 固定 `8000`;宿主机发布端口是 **8010**(`PORT` 可改),
特意避开 8000,留给本机其它 Python 服务。Caddy 反代目标是宿主机 `:8010`。

## 一、DNS(先做)

`megatron.qshunter.top` 的 **A 记录** → VPS 公网 IP。放行防火墙 **80 / 443**。
(Caddy 靠 80/443 的 HTTP-01 质询签 Let's Encrypt 证书,DNS 没生效就签不下来。)

## 二、装 Docker

```bash
curl -fsSL https://get.docker.com | sh
```

## 三、拉代码

```bash
git clone git@github.com:ElectQ/Megatron.git
cd Megatron
# 合并到 master 后直接用 master;未合并前用功能分支:
git checkout feat/phase0-source-ingest
```

## 四、首次部署

```bash
bash deploy.sh
```

会自动:生成 `.env`(随机 session secret、**admin 密码**〔屏幕打印,存好〕、ingest token)、
构建镜像、`alembic upgrade head`、起容器。域名 `megatron.qshunter.top` 已预置进 `.env`。

## 五、填 LLM 密钥后重启

编辑 `.env`,填 `MEGATRON_DEEPSEEK_API_KEY`(要钉钉推送再填 `MEGATRON_DINGTALK_*`),然后:

```bash
bash deploy.sh          # 重新 down/up,加载新 env
```

## 六、HTTPS 入口(Caddy)

仓库自带 `Caddyfile`(已指向你的域名、反代 `localhost:8010`)。装好 Caddy 后:

```bash
caddy run --config ./Caddyfile        # 调试;正式用 systemd 托管
```

完成后:`https://megatron.qshunter.top` = 公开博客前台,`/ui/login` = 后台。
推送里的「查看今日详情」链接会自动用这个域名。

# 数据安全(备份 / 恢复)

**数据都在命名卷 `megatron_data`**(挂到容器 `/app/data`):SQLite 库 `megatron.db`
+ 首启生成的密钥。容器重建/升级(`deploy.sh update`)**不丢**;只有 `deploy.sh clean`
会连卷一起删。

## 备份

```bash
bash deploy.sh backup
# → backups/megatron-YYYYMMDD-HHMMSS.tar.gz
```

备份会**短暂暂停容器**几秒,确保 SQLite(WAL 模式)快照一致,然后自动恢复。

> ⚠️ 本地备份会随磁盘一起死。务必再拷一份到别处:
> `scp backups/*.tar.gz you@another-host:~/megatron-backups/`,或传对象存储。

定时备份(crontab,每天 04:00):

```cron
0 4 * * * cd /path/to/Megatron && bash deploy.sh backup >> backups/cron.log 2>&1
```

## 恢复

```bash
bash deploy.sh restore backups/megatron-YYYYMMDD-HHMMSS.tar.gz
```

会先确认再**覆盖**当前数据,暂停容器 → 还原卷 → 重启。

## 其它安全建议

- `.env` 含密钥,已被 `.gitignore` 忽略,**不要**提交或外传;换机迁移时随备份一起安全拷贝。
- 只用 HTTPS 对外(Caddy)。如无需 IP 直连,可把 `docker-compose.yml` 的端口映射收成
  `127.0.0.1:${PORT:-8010}:8000`,让公网只能经 Caddy 443 进来,`8010` 不暴露。
- 后台 admin 密码可在「系统设置」里改;首启随机密码请立即改成自己的。

# 日常运维

```bash
bash deploy.sh update    # git pull + 重建 + 重启(不丢数据)
bash deploy.sh backup    # 备份
bash deploy.sh logs      # 实时日志
bash deploy.sh status    # 容器状态
bash deploy.sh clean     # ⚠️ 停容器 + 删所有数据(卷、密钥)
```
