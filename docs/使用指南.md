# Megatron 使用指南

基于 MCP 协议的多源情报分析引擎:**数据源采集(MCP / Git)→ LLM 分析 → Webhook 定时推送**。

想最快跑通一次,直接看 [快速上手](快速上手.md)。

## 1. 核心概念

| 概念 | 说明 |
|------|------|
| 数据源(Source) | 数据来源。两类:内置 `twitter`(调度器定时 git 拉取)、MCP 服务器(stdio 本地命令 / SSE 远程端点)。 |
| 采集项(Item) | 从数据源抓取并入库的原始条目,按**采集日**(`collect_date`)归档,`(source, item_id)` 去重。 |
| 提示词模板(Prompt) | 决定 LLM 如何分析,可配结构化输出 schema。 |
| LLM 提供商(Provider) | 模型接入配置(`model` / `api_key` / `api_base` / 温度 / `max_tokens`)。 |
| Webhook 渠道(Channel) | 分析结果的推送目标(如钉钉)。 |
| 任务 / 模块(Task) | 把 `数据源 + 时间窗口 + 过滤 + 提示词 + 模型 + 渠道 + 定时` 组合成一条可运行的流水线。 |
| 运行(Run) | 一次任务执行,状态 `completed` / `failed`,可查看输入条数、耗时、token / 成本、投递结果。 |

## 2. 启动与登录

```bash
megatron init                    # 初始化数据库表
megatron serve dev  --port 8000  # 开发服务器(支持 --reload)
megatron serve prod --port 8000  # 生产服务器
# 或使用 Docker
docker compose up -d
```

其他常用 CLI:

- `megatron seed` — 写入默认提示词模板 + 默认用户 `admin / admin`(仅测试用,请尽快改密)。
- `megatron createsuperuser` — 交互式创建 / 更新登录用户。

访问 <http://localhost:8000> → 跳转 `/ui/login`。

**首次管理员密码**:未设 `MEGATRON_ADMIN_PASSWORD` 时,首启会随机生成一个强密码并**以 WARNING 级别打印到启动日志**(事件 `bootstrap.admin_user_created_with_generated_password`,含明文密码),用户名 `admin`,显示名 `Administrator`。生产环境建议直接用环境变量固定密码。

## 3. 推荐配置顺序

登录后按此顺序一次打通链路:

1. **LLM 提供商** `/ui/providers` — 填 `model` + `api_key`(配 `MEGATRON_MASTER_KEY` 后加密存储)。
2. **提示词模板** `/ui/prompts` — 写分析指令,可配 output schema。
3. **Webhook 渠道** `/ui/channels` — 填 webhook,可「测试」发送一条测试消息。
4. **数据源** `/ui/data/sources` — 见第 4 节。
5. **任务** `/ui/tasks` — 见第 5 节。
6. **运行与查看** `/ui/runs`。

主要页面路由:仪表盘 `/ui/dashboard`、采集数据 `/ui/data/collected`、分析结果 `/ui/data/analyzed`、任务 `/ui/tasks`、运行记录 `/ui/runs`、数据源 `/ui/data/sources`、提示词 `/ui/prompts`、提供商 `/ui/providers`、渠道 `/ui/channels`、定时 `/ui/schedules`。

## 4. 数据源

### A. 内置 Twitter / Git 拉取(开箱即用)

- 设环境变量 `MEGATRON_SOUNDWAVE_REPO_URL` 指向 Soundwave 数据仓库(接受完整 GitHub URL 或 `owner/repo`;缺省回退到内置 `ElectQ/Soundwave`)。
- 调度器每天 **06:00 UTC** 自动增量拉取(watermark 增量,只拉新日期),入库标签 `source=twitter`。
- 任务里数据源选 `twitter` 即可,**无需**在此页添加。

### B. MCP 服务器(本次打通的路径)

点「添加 MCP 服务器」:

- **服务器名称**:≤32 字符,**不能用保留名** `twitter` / `soundwave` / `mcp`。这个名字就是任务里可选的数据源名。
- **传输方式**:
  - **SSE(远程端点)**:在地址栏填完整 URL(如 `https://host/mcp`)。
  - **stdio(本地命令)**:在命令行填 Megatron 本地要启动的命令(如 `python -m mcp_servers.soundwave --repo owner/repo --branch master`);**留空则使用内置 soundwave 服务器**。
- **资源过滤**:可选。

操作建议:

1. 填完先点 **「测试连接」**(保存前测试,不写库)。成功会列出该 MCP 服务器暴露的工具。
2. 确认无误后 **「连接 MCP 服务器」** 保存。保存后自动生成同名数据源配置。
3. 列表里每行还有 **Test / Discover**:Test 用真实协议连一次并回写状态;Discover 用真实协议 `list_tools()` 枚举工具并持久化 capabilities。

## 5. 任务

- **数据源**:下拉包含 `twitter` + 已启用的 MCP 源名 + 内置类型 + 现有任务用过的源(打开旧任务不会丢选)。
- **数据源引用(source_ref)**:可选二级过滤(如某个 list_id),留空 = 全部。
- **时间窗口(time_mode)**:
  - `today` — 今天(UTC)采集的;
  - `date` — 指定某天(`target_date`);
  - `range` — 日期区间;
  - `rolling` — 最近 N 小时(`window_hours`,按发布时间)。
  > MCP 源运行时会先增量抓取,并自动把窗口对齐到本次抓到的最新采集日。
- **过滤器 / max_items**:可选,控制送入 LLM 的条目数量。
- **提示词模板 / LLM 提供商 / Webhook 渠道**:选前面配置好的。
- **定时(cron)**:如 `0 7 * * *`(每天 07:00 UTC);留空 = 仅手动运行。

## 6. 运行结果如何判断

- **completed(有内容)**:抓取 → 分析 → 投递走完。
- **completed / 无数据**:数据源可达但确实没有符合时间窗口的新数据(正常空跑)。
- **failed**:采集 / 连接 / 协议 / 分析出错。**错误明确记录在运行的 error 字段**——MCP 连不上、命令跑不起来、返回解析失败,都会让这次 run 变 FAILED,不再伪装成"无数据"的假成功。
- **投递**:某个渠道失败**不影响**分析本身的 completed 状态,失败明细记录在投递日志(DeliveryLog),单个坏渠道不会拖垮整批。

## 7. 增量与去重机制

- 每个数据源维护一个 **watermark(最后成功采集的日期)**。MCP 源运行时只抓取 watermark 之后的新日期;成功入库后推进 watermark。
- 入库按 `(source, item_id)` 唯一去重,重复运行不会产生重复数据。
- 冷启动(无 watermark)会抓取全部可用日期,后续转为增量。

## 8. 环境变量清单

> 全部使用 `MEGATRON_` 前缀。`admin_token` / `session_secret` 也可分别放到项目根的 `.admin_token` / `.session_secret` 文件里。

| 变量 | 用途 |
|------|------|
| `MEGATRON_ADMIN_PASSWORD` | 管理员密码(不设则首启随机生成并打印日志) |
| `MEGATRON_ADMIN_TOKEN` | 管理 API Token(默认值仅供开发,生产必改) |
| `MEGATRON_SESSION_SECRET` | 会话签名密钥(生产必改) |
| `MEGATRON_MASTER_KEY` | 密钥加密主密钥(用于加密 `api_key` / 敏感配置;不设则兼容明文) |
| `MEGATRON_DATABASE_URL` | 数据库连接串(默认 SQLite `sqlite+aiosqlite:///./megatron.db`;支持 Postgres) |
| `MEGATRON_BASE_URL` | 站点基础 URL(默认 `http://localhost:8000`) |
| `MEGATRON_SOUNDWAVE_REPO_URL` | 内置 twitter / git 拉取的数据仓库地址 |
| `MEGATRON_INGEST_TOKEN` | 推送式采集接口 Token |
| `MEGATRON_ENV` | 运行环境(`development` / 生产) |

## 9. 常见问题排错

- **任务跑出"无数据"**:确认数据源已成功采集(看 `/ui/data/collected`),且任务时间窗口覆盖了这些数据的采集日;MCP 源要先能「测试连接」通过。
- **MCP 测试连接失败**:stdio 检查命令能否在服务端环境直接跑通(依赖、路径);SSE 检查 URL 可达与鉴权;错误信息会返回在测试结果里。
- **stdio 想用内置服务器**:命令行留空即可,回退到内置 soundwave。
- **SSE 本地验证不了**:内置 soundwave 只支持 stdio;SSE 需对接真正支持 SSE 的远程 MCP 服务器。
- **老的 twitter 任务照常工作**:git-pull 路径未改动,由调度器定时采集,任务运行时不做内联抓取。
