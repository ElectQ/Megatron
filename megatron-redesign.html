Design proposal · Cloudflare-style

Megatron, rebuilt around the pipeline it actually runs

A redesign of the admin UI in the Cloudflare vernacular: orange reserved strictly for the primary action and your current location, warm-neutral surfaces everywhere else, and navigation that mirrors the real data flow — Source → Collect → Analyze → Deliver. Everything below is a live mock; toggle the theme, top-right.

1 · Navigation & the Overview page

Left nav is grouped into four sections that read as the pipeline. Orange marks only the active item. The header carries a breadcrumb + one primary action; secondary views live in in-page tabs, not more sidebar entries.

3 · Navigation order & one name per thing

Today the same concept has up to three names across nav, routes and code. Pick one human word per concept and use it everywhere the user can see. Keep the code name internal.

Ordering rationale. Sidebar order = order you set the system up + order data flows: connect a Source, watch it Collect, write a Prompt + pick a Model, wire a Task to Deliver on a Schedule. Config lives at the bottom because you touch it once; Data/Analysis live at top because you watch them daily.
